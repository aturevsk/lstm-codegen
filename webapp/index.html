LSTM Seq-to-Seq C Code Generation

Four approaches to deploying a PyTorch LSTM model as bare-metal C on STM32F746G-Discovery. Benchmarked with Apple Clang 17 (gcc) at -O3 -ffast-math — 7 × 1000-run trials, seed-42 input. All four implementations produce identical predictions (100% match).

Key Findings

  • 🏆Option 1 (hand-written C) is fastest at 53.6 µs mean. Gate-split weights, combined bias, inline activations with ARM FPU pragmas.
  • 🤖Options 2–4 use C actually generated by MATLAB Coder — no hand-written inference code. Fully automatic from model to deployable C.
  • 📊MATLAB Coder options are 1.63–1.74× slower due to general-purpose code generation overhead vs. hand-tuned implementation.
  • 💾Flash footprint: Opt 1 = 12 KB vs Opts 2–4 = 210–215 KB (17–18× larger — MATLAB Coder runtime + weights).
  • ✅All four options produce 100% identical predictions on the 75-timestep seed-42 test sequence.
  • 🎯STM32 estimates (×1.10 factor): Opt 1 ≈ 59 µs, Opts 2–4 ≈ 96–103 µs @ 216 MHz.

Option 1 — Hand-Written C

Gate-split weight matrices stored in flash ROM. Combined bias (bias_ih + bias_hh merged at export). Inline sigmoid/tanh helpers mapped to FPU instructions. ARM-specific pragmas for -O3 and FPv5-D16.

⚡ 53.6 µs  ·  12 KB flash

Option 2 — MATLAB Coder + PyTorch Pkg

Loads .pt2 via loadPyTorchExportedProgram. MATLAB Coder applies MLIR/TOSA lowering automatically. Single-file codegen output (lstm_infer_opt2.c ~213 KB). No external deep learning libraries.

87.5 µs  ·  210 KB flash

Options 3 & 4 — dlnetwork

Import via PyTorch (.pt2) or ONNX, strip custom argmax layer, rebuild as standard dlnetwork. MATLAB Coder with DeepLearningConfig('none') generates ~11 C files. networkParams*.bin holds binary weights.

~93 µs  ·  215 KB flash

Option 1 — Hand-Written C

LSTM Cell — Key Equations

// Gate pre-activations
i_t = sigmoid( Wi_i·x + Wh_i·h + b_i )
f_t = sigmoid( Wi_f·x + Wh_f·h + b_f )
g_t = tanh   ( Wi_g·x + Wh_g·h + b_g )
o_t = sigmoid( Wi_o·x + Wh_o·h + b_o )
// State update
c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t
h_t = o_t ⊙ tanh(c_t)
// Classification
output_t = argmax( FC·h_t + b_fc )

Option 2 — MATLAB Coder + PyTorch Support Package

Option 3 — MATLAB Coder via importNetworkFromPyTorch → dlnetwork

Imports the .pt2 file using importNetworkFromPyTorch, strips the custom auto-generated argmax layer (not codegen-compatible), and rebuilds a standard dlnetwork from SequenceInput + LSTM + FC layers. MATLAB Coder with DeepLearningConfig('none') generates ~11 C files. Binary network parameters are stored in networkParamsInfo_net3*.bin files loaded at runtime.

Option 4 — MATLAB Coder via ONNX → importNetworkFromONNX → dlnetwork

Exports the PyTorch model to ONNX format, imports via importNetworkFromONNX, and explicitly reconstructs a dlnetwork using weights extracted from ONNX learnables (FC bias sourced from PT2 reference). Identical codegen path to Option 3 — confirms ONNX is a viable alternative input route with identical performance.
